# AEGIS Project Rules

## General

Never generate placeholder code.

Never use any.

Never disable TypeScript.

Never disable ESLint.

Never use inline CSS.

Never use magic numbers.

Always write reusable code.

---

## Components

Maximum 200 lines.

One responsibility only.

Use functional components.

Use TypeScript.

Use named exports.

---

## Styling

Tailwind only.

No inline styles.

No CSS modules.

Use design tokens.

---

## API

Axios only.

Never fetch directly inside components.

API calls belong in services.

---

## State

React Query

Server data

Zustand

Global UI

React State

Component state

---

## Forms

React Hook Form

Zod validation

## Git

Meaningful commit messages.

Small commits.

No commented code.

No console.log in production.

---

## Performance

Dynamic imports where appropriate.

Lazy load heavy components.

Optimize images.

Avoid unnecessary re-renders.
